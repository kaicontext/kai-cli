How it works

Live Sync

Multiple AI agents working on the same codebase see each other's changes in real time via SSE. Edits are merged semantically — two agents editing different functions in the same file merge cleanly. No commits, no pull, no conflicts.

Semantic Graph

Tree-sitter AST parsing extracts functions, classes, call chains, imports, and test coverage into a local graph database. Kai understands what code means, not just what lines changed. This is what makes semantic merging possible.

Semantic 3-Way Merge

When two agents edit the same file, Kai merges at the function level, not the line level. It knows that adding a parameter to validateToken and adding a new function refreshToken are independent changes. Real conflicts — two agents changing the same function differently — are flagged precisely.

AI Authorship Tracking

Every AI edit is checkpointed with agent name, model, and line ranges. kai blame shows exactly which code was AI-generated vs human-written. Know who wrote what — human or machine.

2x Token Efficiency

kai prime queries the semantic graph in 188ms and pre-loads the right files into the AI's context. 53% fewer tokens, 42% lower cost. The agent reasons instead of exploring.

Selective CI

Run only the tests affected by your changes. The semantic graph knows which tests cover which functions — no more running the full suite.
